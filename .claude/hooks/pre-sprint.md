# hook: pre-sprint — Chuẩn bị sprint

Trước khi bắt đầu sprint, sinh:
1. Implementation plan (theo backlog P0/P1/P2 của sprint).
2. Risk analysis (rủi ro kỹ thuật/nghiệp vụ + giảm thiểu).
3. Dependencies (giữa item, với sprint trước, với provider ngoài).
4. Affected files (danh sách dự kiến chạm, theo từng tầng).

Output: `PLAN.md`. Không bắt đầu code trước khi có PLAN.md và đã chạy `pre-task`.
